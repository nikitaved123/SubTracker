# Трекер подписок

## Актуальность
Многие современные сервисы работают по подписке. Со временем люди забывают о некоторых из них, и сервисы продолжают списывать деньги за функции, которые уже не используются. Поэтому возникает необходимость в контроле и отслеживании подписок.

## Целевая аудитория
Сервис рассчитан как на молодых людей от 16 лет, так и на взрослых до 50 лет, которые активно пользуются цифровыми сервисами. Он также будет полезен тем, кто хочет оптимизировать свой бюджет или просто контролировать расходы.

## Предполагаемый функционал
- Пользователь самостоятельно вводит информацию о подписках: название, дату начала подписки и дату ежемесячного списания денег.
- Сервис показывает общую сумму расходов на все подписки за месяц, а также сумму по каждой отдельной подписке.
- Сервис заранее предупреждает пользователя о скором списании средств.

## Используемые технологии (предварительно)

### Фронтенд
**HTML5, CSS3 (Tailwind CSS), JavaScript** — современный адаптивный интерфейс, лёгкость реализации.

### Бэкенд
**Python / FastAPI** — высокая производительность, простота разработки API.

### База данных
**PostgreSQL** (или SQLite для MVP) — надёжное хранение данных подписок и пользователей.

### Аутентификация
**JWT + хеширование паролей (bcrypt)** — безопасность пользовательских данных.

### Визуализация
**Chart.js или ECharts** — лёгкое подключение графиков и диаграмм.

### Хранение данных
**localStorage** (для MVP без бэка) или **PostgreSQL** — возможность простой версии для демонстрации.

## Ожидаемый результат
Готовый к использованию продукт, который позволяет пользователю контролировать регулярные платежи и выявлять неиспользуемые подписки. В рамках проекта также разработана документация: пояснительная записка (30–40 стр.), руководство пользователя и презентация.
